低电平触发

GPIO的功能就是用来控制高低电平的，而大部分外设都是通过高低电平来工作的，所以将GPIO与各种外设联系起来实现各种功能。

在stm32单片机中引脚分成了很多组GPIOA~GPIOE，并且每一组中又有0-15共16个引脚;每个IO口（引脚）由4bit位来控制那么一组IO就需要16*4 = 64bit,而8个引脚就已经32位，构成一个寄存器了，所以使用两个寄存器CRL和CRH来控制GPIO的16个引脚，高32bit由GPIOX_CRH寄存器来配置;低32bit由GPIOX_CRL寄存器来进行配置.对应的引脚CRL为0到7，CRH为8到15。从上面我们可以看出不管是GPIOX_CRL还是GPIOX_CRH每个IO都是由CNF[1:0]和MODE[1:0]两部分进行配置选择的;其中MODE[1:0]是进行配置输入还是输出，一般00为输入， 11为输出。CNF[1:0]则配合MODE[1:0]进行具体的模式配置。所以如果使用8-15号引脚就是用CRH，否则使用CRL，使用低寄存器中的某个引脚就控制对应的引脚的CNF和MODE

GPIO输入

一般配置为上拉输入模式

上拉输入：有上拉电阻

按键未按下时端口接高电平，即高电平 1 状态—3.3v

按键按下时端口接低电平，即低电平 0 状态——-0v

下拉输入：有下拉电阻

按键未按下时端口接低电平，即低电平 0 状态—0v

按键按下时端口接高电平，即高电平 1 状态——3.3v

1、 上拉输入（GPIO_Mode_IPU）

输入上拉就是直接从IO端口进来，通过上拉电阻将电平拉高，再经过施密特触发器转换成0、1信号，读取此时的引脚电平为高电平, 然后就输入到数据到寄存器里面。

2、 下拉输入（GPIO_Mode_IPD）

输入下拉就是直接从IO端口进来，通过下拉电阻将电平拉低，再经过施密特触发器转换成0、1信号，读取此时的引脚电平为低电平, 然后就输入到数据到寄存器里面。

3、 模拟输入（GPIO_Mode_AIN）

信号进入后不经过上拉电阻或者下拉电阻，关闭施密特触发器，经由另一线路把电压信号传送到片上外设模块。比如传送给ADC模块，由ADC采集电压信号。所以可以理解为模拟输入的信号是未经处理的信号。

4、 浮空输入(GPIO_Mode_IN_FLOATING)

浮空最大的特点就是电压的不确定性，它可能是0V，也可能是VCC，还可能是介于两者之间的某个值（最有可能） 浮空一般用来做ADC输入用，这样可以减少上下拉电阻对结果的影响。由于其输入阻抗比较大，一般把这种模式用于标准的通讯协议，比如IIC、USART的等。

TIM（Timer）定时器，定时器可以对输入的时钟进行计数，并在计数值达到设定值时触发中断，如果对72MHz计72个数，那就是1MHz即1us的时间；如果计72000个数，那就是1KHz即1ms的时间

计数器：用来执行计数定时的一个寄存器，可以对预分频后的计数时钟进行计数，计数时钟每来一个上升沿计数器的值就+1，计数器也是16位的，里面的值可以从0~65535

预分频器：可以对计数器的时钟进行分频，让这个计数更加灵活

​ 写0：不分频或者一分频，输出频率=输入频率=72MHz；

​ 写2：三分频，输出频率=输入频率/3=24MHz；

​ 概括归纳：实际分频系数 = 预分频器的值 + 1

功能2：定时器输出比较的功能，最常见的用途就是产生PWM波形，用于驱动电机等设备

实际上就是用来输出一个模拟信号，那么它是如何实现的呢？就是来调制脉冲的宽度，通过输出比较寄存器，当输出比较器中的某一值大于另一个值时，触发上升沿，反之下降。亮度如何调节呢？就是通过快速的开启关闭，在这期间由个惯性，不会迅速变化，而是逐渐变化，此时通过占空比来计算亮暗程度，如左图，当高电平占比越大，可以用占空比越大表示，低电平占比越大，占空比越小，可以以此等效为模拟信号。一个区间，若高电平占80%,则表示占空比为80%，占空比等效为模拟电压的大小。若高电平为5v，低电平为0v，若占空比为50%，则表示模拟电压为2.5v。因此可以实现点击控速。

计算公式：

PWM频率：Freq = CK_PSC / (PSC + 1) / (ARR + 1)

PWM占空比：Duty = CCR / (ARR + 1)

PWM分辨率：Reso = 1 / (ARR + 1)——占空比最小的变化步距，值越小越好

案例：实现使用计时器实现LED等的亮灭。

思路：首先实现计时一秒：因为一个代表1us，65535就是65毫秒，那么怎么使用1ms呢？从64535处开始计时，到65535就是1000个us就是1ms（开始执行中断函数），那么如何计时1s呢，利用if函数判断该函数是否被调用了1000次就是1s

脉冲的原理：默认是共有65535个脉冲，每个脉冲1us，相当于所有脉冲已经编号号并且固定好，当执行的时候仅需要设置从哪里开始执行即可，达到第65535个脉冲是开始执行中断程序，执行完后默认从0开始，而不是从设置的起点开始。

假如说，我们定时50毫秒，TH0，TL0对应着什么值呢？ 上面我们说了，TH0，TL0，组成的16位计数器计数范围是 0—65535 。50ms = 50 000us，我们只要让 TH0 TL0 从（65535 - 50000）开始计数，TH0，TL0就会不停的+1，直到TH0，TL0=65535，就是计数了50000次，时间就是过了50ms。我们只要在中断程序里面，重新赋值给TH0，TL0=(65535 - 50000)，定时器就会不停得每50ms中断一次了。

Blazor Server 托管模型具有以下优点：（网页文件都是cshtml类型，因为是服务端渲染）

  • 下载项大小明显小于 Blazor WebAssembly 应用，且应用加载速度快得多。
  • 支持瘦客户端。 例如，Blazor Server 应用适用于不支持 WebAssembly 的浏览器以及资源受限的设备。
  • 应用的 .NET/C# 代码库（其中包括应用的组件代码）不适用于客户端。

Blazor Server 托管模型具有以下局限性：

  • 通常延迟较高。 每次用户交互都涉及到网络跃点。
  • 不支持脱机工作。 如果客户端连接失败，应用会停止工作。

Blazor WebAssembly 托管模型具有以下优点：（网页文件都是razor类型，因为是客户端渲染）

  • 没有 .NET 服务器端依赖，应用下载到客户端后即可正常运行。
  • 工作可从服务器转移到客户端。

Blazor WebAssembly 托管模型具有以下局限性：

  • 应用仅可使用浏览器功能。

  • 需要可用的客户端硬件和软件（例如 WebAssembly 支持）。

  • 下载项大小较大，应用加载耗时较长。
